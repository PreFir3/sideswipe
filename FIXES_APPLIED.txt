  Hand tracking turns on but features don't work reliably, loses tracking easily,
  and is overly sensitive to accidental movements.

  ✅ MediaPipe confidence thresholds too low (0.6-0.7)
  ✅ Gesture detection parameters too lenient
  ✅ Insufficient stabilization/confirmation frames
  ✅ Poor handling of invalid hand detections

================================================================================
                            DETAILED CHANGES
================================================================================

CHANGE 1: MediaPipe Detection Confidence
  IMPACT: Only high-confidence hands are tracked, weak detections eliminated
  
  BEFORE: 0.70 (70% confidence acceptable)
  AFTER:  0.85 (85% confidence required) ← +21% MORE STRICT

CHANGE 2: MediaPipe Tracking Confidence
  IMPACT: Hand must maintain tracking confidence to stay tracked
  
  BEFORE: 0.60 (60% tracking acceptable)
  AFTER:  0.80 (80% tracking required) ← +33% MORE STRICT

CHANGE 3: OK Hand Circle Threshold
  IMPACT: Thumb and index must be much closer to form circle
  
  BEFORE: 0.25 (25cm separation okay)
  AFTER:  0.04 (4cm separation required) ← 6X MORE STRICT!
  
CHANGE 4: OK Hand Confirmation
  IMPACT: Gesture must be held longer to avoid false positives
  
  BEFORE: 3 frames (0.1 seconds at 30fps)
  AFTER:  15 frames (0.5 seconds at 30fps) ← 5X LONGER!

CHANGE 5: Swipe Minimum Movement
  IMPACT: Requires more hand movement to trigger
  
  BEFORE: 100 pixels
  AFTER:  150 pixels ← +50% MORE MOVEMENT REQUIRED

CHANGE 6: Swipe Confirmation
  IMPACT: Direction must be confirmed more frames
  
  BEFORE: 3 frames (quick)
  AFTER:  6 frames (slower) ← 2X MORE CONFIRMATION

CHANGE 7: Number Stabilization
  IMPACT: Finger count must be stable longer
  
  BEFORE: 10 frames (0.33 seconds)
  AFTER:  20 frames (0.67 seconds) ← 2X LONGER HOLD

CHANGE 8: Hand Filtering
  IMPACT: Invalid hand detections are pre-filtered
  
  BEFORE: Process hand_detections[0] if present
  AFTER:  Filter to detected_hands = [h for h in hand_detections if h.present]

================================================================================
                         HOW TO VERIFY FIXES WORK
================================================================================

TEST 1: Hand Detection (Enable debug mode with 'd')
  PASS:  Hand skeleton appears only when hand visible, no jitter
  FAIL:  Hand skeleton flickers or appears randomly

TEST 2: OK Hand Gesture
  PASS:  Make OK hand, hold 0.5s, system toggles ON/OFF
  FAIL:  Triggers randomly or requires strange hand position

TEST 3: Swipe Gesture
  PASS:  Clear 150px swipe switches tabs smoothly
  FAIL:  Jittery or accidental swipes trigger

TEST 4: Number Selection
  PASS:  Hold 2-5 fingers steady 0.7s, counts correctly
  FAIL:  Counts jitter or wrong numbers appear
